# Scriptor Scenario Registry

Use this file before and after bug fixes that touch extension state, Google Docs word counts, document binding, catch-up, sessions, or persistence. Each critical scenario should have happy-path, edge-path, and failure-path coverage where practical. If a scenario is not automated, keep its manual fallback in `QA_CHECKLIST.md`.

Coverage values:

- Automated: an existing test covers the exact behavior.
- Partially automated: tests cover nearby logic, but not the full user journey.
- Manual only: covered by `QA_CHECKLIST.md` and not automated yet.
- Missing: no meaningful automated or manual coverage yet.
